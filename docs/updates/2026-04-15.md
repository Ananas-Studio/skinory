- `git push origin main` ile deploy tetiklendi ancak Azure login adımında fail oldu.

- Yeni SP oluşturuldu (`az ad sp create-for-rbac`), `AZURE_CREDENTIALS` GitHub secret güncellendi.
- ACR login da fail etti — `ACR_NAME` ve `ACR_LOGIN_SERVER` secret'ları yanlıştı, SP'ye `AcrPush` rolü eksikti.
- Tüm secret'lar düzeltildi, SP'ye ACR push yetkisi verildi → deploy başarılı.

## 2. Backend Scalability & Cost Control Overhaul

10.000+ eşzamanlı kullanımda stabil, öngörülebilir maliyetli ve güvenli bir sistem hedefiyle 12 root-cause risk tespit edildi ve kökten çözüldü.

### A) Rate Limiting & Koruma Katmanı
- **Global rate limiter** eklendi: IP başına 100 req/dk.
- **Strict rate limiter** eklendi: pahalı endpoint'ler için 10 req/dk.
- **Auth rate limiter** eklendi: login/register için 20 req/dk.
- `express.json({ limit: "1mb" })` ile body size sınırı konuldu.
- Dosyalar: `rate-limit.middleware.ts`, `app.ts`

### B) Usage & Kota Mimarisi (Race Condition Fix)
- `checkAndRecordUsage()` fonksiyonu atomik hale getirildi — tek SQL `INSERT...SELECT...WHERE count < limit` ile race condition kaldırıldı.
- `getUsageSummary()` artık tüm satırları çekip JS'te saymıyor; `GROUP BY COUNT(*)` ile DB aggregation yapıyor.
- 4 yeni usage kategorisi eklendi: `scan_ocr`, `scan_image_recognize`, `social_scrape`, `social_enrich`.
- Dosyalar: `usage.service.ts`, `usage.middleware.ts`, `packages/core/src/subscription.ts`, `db-types.ts`

### C) AI Chat Maliyet Kontrolü
- Her istekte tüm mesaj geçmişi gönderilmesi durduruldu.
- **Sliding window** (son 10 mesaj) + **otomatik özet** (eski mesajlar için) yaklaşımı uygulandı.
- `SUMMARY_CHAR_BUDGET = 2000` karakter bütçesi konuldu.
- Token telemetry loglanıyor (prompt_tokens, completion_tokens).
- Dosya: `advice.service.ts`

### D) Sosyal Enrich Fan-Out Kontrolü
- Request body'de max 10 ürün sınırı (`z.array().max(10)`).
- Metin alanı max 10.000 karakter.
- OBF aramaları için `Semaphore(3)` concurrency limiti + `TtlCache(5dk)`.
- `social_scrape` ve `social_enrich` endpoint'lerine usage check eklendi.
- Dosyalar: `social-product-enricher.ts`, `social.route.ts`

### E) Görsel İşleme Dayanıklılığı
- `concurrencyGuard` middleware'i: max 3 eşzamanlı görsel işlem, kuyruk > 10 ise 503.
- Scan endpoint'lerine (`/ocr`, `/recognize`, `/evaluate`) strict limiter + usage check + concurrency guard eklendi.
- Dosyalar: `concurrency.middleware.ts`, `scan.route.ts`, `semaphore.ts`

### F) OpenAI Timeout'ları
- Tüm 5 OpenAI çağrı noktasına `AbortSignal.timeout(30_000)` eklendi.
- Dosyalar: `evaluation.service.ts`, `image-recognition.service.ts`, `ocr/vision-provider.ts`, `social-product-detector.ts`, `advice.service.ts`

### G) Pagination & Veri Erişim
- Limitsiz list endpoint'lerine pagination eklendi: `inventory/items`, `advice/sessions`, `advice/messages`, `favorites/ids`.
- Varsayılan limit + max limit zorunlu hale getirildi.
- Backward compatibility korundu: `data` alanı dizi olarak kaldı, pagination metadata ayrı `pagination` alanında.
- Dosyalar: `inventory.route.ts`, `advice.route.ts`, `favorites.route.ts`

### H) Index & Migration Altyapısı
- `pg_trgm` extension + GIN trigram index (`products.name`) eklendi.
- 4 composite index eklendi: `scans(user_id, result_status, created_at)`, `advice_sessions(user_id, created_at)`, `products(is_active, category)`, `inventory_items(inventory_id, status)`.
- Minimal SQL migration runner oluşturuldu (`_migrations` tablosu ile tracking).
- Dosyalar: `migrations/001-indexes-and-trgm.sql`, `migrations/runner.ts`

### I) Connection Pool
- Sequelize pool config: `max: 20, min: 5, idle: 10000, acquire: 30000`.
- Dosya: `config/database.ts`

## 4. Pagination Backward Compatibility Fix

- Pagination değişiklikleri `data` alanını obje olarak wrap etmişti (`{ items, total, limit, offset }`).
- Frontend `data`'nın düz dizi olmasını bekliyordu → `S.map is not a function` hatası.
- **Fix:** `data` düz dizi olarak bırakıldı, pagination metadata ayrı `pagination` alanına taşındı.

## Kalan Riskler & Sonraki Adımlar

1. **Multi-instance scaling:** Rate limit ve cache in-memory — birden fazla replica'da Redis'e geçiş gerekir.
2. **Auth limiter mount:** `authLimiter` oluşturuldu ama auth route'larına henüz mount edilmedi.
3. **Integration test:** Pahalı endpoint'lerde usage + rate limit birlikte çalışma testi yazılabilir.
4. **Trigram query optimizasyonu:** GIN index oluşturuldu, ILIKE zaten bundan faydalanıyor ama explicit `similarity()` fonksiyonu ile daha iyi sonuç alınabilir.
